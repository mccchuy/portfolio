Our task was to build an intrepid robot that would travel across the Science and Engineering Complex at Tufts.

Requirements

The robot in action
The robot must do three things in 15 minutes:
  • High-five a teacher's assistant
  • Travel upstairs to Prof. Jason Rife's office
  • High-five Prof. Rife


Restrictions:
  • Robot should be remote controlled
  • Robot should be controlled through wifi through a laptop or phone
  • Robot cannot fly (no drones)
  • Cannot clear a path for the robot (must travel over multiple surfaces)

How it works

Raspberry Pi Setup

In order for the robot to run remotely, we needed Raspberry Pi to connect to the Wifi.

We used their built-in Python program to write the code that would be burned onto the Raspberry Pi, then sent to the Arduino, which would control the motors on either side of the robot through the motor driver (L298N).
Console set-up to burn the Raspberry Pi's SD card

Arduino Setup

Motor driver (L298N) and Arduino set-up
The Raspberri Pi remotely sends messages to the Arduino, which then triggers a pin response to go either HIGH or LOW on the four selected pins. These pins correspond to the L298N motor driver, which will trigger an output of voltage to each of the stepper motors on either side of the robot.
